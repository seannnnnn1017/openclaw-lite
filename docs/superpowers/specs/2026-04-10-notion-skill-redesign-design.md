# Notion Skill Redesign ‚Äî Design Spec

**Date:** 2026-04-10  
**Branch:** refactor/agent-modular-structure  
**Sandbox:** `sandbox_for_claude/SKILL/`  
**Test target page:** `https://www.notion.so/33e5aafddb3b80daae72c72ecf916479`

## Problem Summary

Four debug sessions (2026-04-10) revealed five root-cause failures when the agent was asked to copy local Markdown files into Notion sub-pages:

| # | Symptom | Root Cause |
|---|---------|------------|
| 1 | Agent cannot list folder contents | `file_tool.py` has no `list_directory` action; `read(dir)` returns "Only file paths are supported" |
| 2 | Agent hallucinates `__delegate__` / `delegate` on file-control | SKILL.md boundary rules are missing |
| 3 | Agent calls `API-create-page` (non-existent) | `API-post-page` example for sub-pages is absent from SKILL.md |
| 4 | Agent does not know how to create sub-pages | No `parent.page_id` example; only `parent.database_id` shown |
| 5 | Batch tasks fail after ~20 steps | No cross-skill workflow SOP; agent tries to read local files inside the Notion delegate |

---

## Solution: Option B ‚Äî Two-skill patch + new notion_workflow skill

All changes are developed and tested in `sandbox_for_claude/SKILL/`.  
Production skills (`agent/SKILLs/`) are not modified until sandbox tests pass.

## Component 1 ‚Äî file_control additions

**File:** `sandbox_for_claude/SKILL/file_control/scripts/file_tool.py`

### New action: `list_directory`

| Field | Detail |
|-------|--------|
| Input | `path` (str, required) ‚Äî directory path |
| Input | `pattern` (str, optional, default `"*"`) ‚Äî glob pattern |
| Input | `recursive` (bool, optional, default `false`) |
| Output | `data.files`: list of `{name, path, size_bytes, extension}` |
| Output | `data.count`: total file count |
| Error | Returns structured error if path is not a directory or does not exist |

### New action: `read_all`

| Field | Detail |
|-------|--------|
| Input | `paths` (list[str], optional) ‚Äî explicit file paths |
| Input | `dir` (str, optional) ‚Äî directory; combined with `pattern` |
| Input | `pattern` (str, optional, default `"*.md"`) ‚Äî used with `dir` |
| Input | `encoding` (str, optional, default `"utf-8"`) |
| Output | `data.files`: list of `{path, name, content, status, error?}` |
| Output | `data.success_count`, `data.error_count` |
| Safety | Single file > 200 KB: content truncated, `truncated: true` flag set |
| Safety | One file failure does not abort the rest |

## Component 2 ‚Äî notion_basic fixes

**Files:** `sandbox_for_claude/SKILL/notion_basic/SKILL.md` and `examples.md`

### SKILL.md additions

1. **Hallucination guard** ‚Äî explicit rule: there is no `delegate`, `__delegate__`, or `API-create-page` action. Violating calls return an error.

2. **Sub-page creation example** (the missing piece from session_182538):
```json
{"skill":"notion-basic","action":"tools/call","args":{
  "name":"API-post-page",
  "arguments":{
    "parent":{"page_id":"<parent_page_id>"},
    "properties":{"title":{"title":[{"text":{"content":"Sub-page Title"}}]}}
  }
}}
```

3. **Block append example** ‚Äî correct `API-patch-block-children` shape with `paragraph`, `heading_1`, `code` block types.

4. **Batch task SOP** ‚Äî explicit numbered steps for "read local files ‚Üí create Notion pages":
   - Step 1: use `file-control read_all` to obtain file contents
   - Step 2: use `notion-basic tools/call API-post-page` to create parent page
   - Step 3: loop ‚Äî for each file, `API-post-page` (sub-page) then `API-patch-block-children` (content)
   - Rule: never attempt to read local files from inside a notion-basic delegate call

### examples.md additions

- Sub-page creation under a known `page_id`
- `API-patch-block-children` with multi-block payload
- Multi-step batch example (abbreviated)

## Component 3 ‚Äî notion_workflow (new skill)

**Location:** `sandbox_for_claude/SKILL/notion_workflow/`

### Folder structure

```
notion_workflow/
  SKILL.md
  examples.md
  skills_config.json
  scripts/
    notion_workflow_tool.py
  test_results/
    (auto-generated by test runner)
```

### Supported actions

| Action | Description |
|--------|-------------|
| `import_folder` | List dir ‚Üí read all ‚Üí create parent page ‚Üí create sub-pages with content |
| `import_files` | Same as import_folder but takes explicit `paths` array |
| `batch_create_pages` | Takes `[{title, content, parent_id}]` array; creates pages in order |
| `append_content` | Appends Markdown text to an existing page; auto-chunks at 100 blocks |
| `sync_folder` | Like import_folder but skips sub-pages whose title already exists under parent |

### `import_folder` execution flow

```
1. list_directory(path, pattern="*.md")
2. read_all(paths=[...all found files...])
3. API-post-page ‚Üí create parent page (parent: caller-supplied page_id)
4. for each file in results:
     a. API-post-page ‚Üí create sub-page (parent: parent page id from step 3)
     b. markdown_to_blocks(content) ‚Üí chunked block lists
     c. for each chunk:
          API-patch-block-children ‚Üí append blocks to sub-page
5. Return summary: {parent_page_id, parent_url, pages: [{title, page_id, url, status}], success_count, error_count}
```

### Markdown-to-blocks conversion rules

| Markdown element | Notion block type |
|-----------------|-------------------|
| `# Heading` | `heading_1` |
| `## Heading` | `heading_2` |
| `### Heading` | `heading_3` |
| Paragraph text | `paragraph` |
| ` ```code``` ` | `code` (language detected from fence) |
| `- list item` | `bulleted_list_item` |
| `1. list item` | `numbered_list_item` |
| `> quote` | `quote` |
| `---` | `divider` |

Chunk size: 90 blocks per `API-patch-block-children` call (buffer below Notion's 100 limit).

### `sync_folder` deduplication logic

1. `import_folder` step 1‚Äì2 (list + read)
2. `API-get-block-children(parent_page_id)` ‚Üí get existing child page titles
3. Skip files whose stem matches an existing child title
4. Create only new files

### skills_config.json

```json
{
  "skills": [{
    "name": "notion-workflow",
    "enabled": true,
    "execution_mode": "invoked",
    "path": "notion_workflow",
    "tool": {
      "type": "python_function",
      "module": "sandbox_for_claude.SKILL.notion_workflow.scripts.notion_workflow_tool",
      "function": "run"
    }
  }]
}
```
